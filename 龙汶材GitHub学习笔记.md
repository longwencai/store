# GitHub学习笔记

### 1、认识Git和GitHub

- Git：开源的分布式版本控制系统，负责在本地跟踪文件更改、记录历史版本，支持离线工作，是代码管理的核心工具。

- GitHub：基于 Git 的云平台，提供远程代码托管服务，额外支持协作、代码审查、自动化等功能，是开发者的协作与展示平台。

### 2、核心概念解析

1. 仓库（Repository）：存储项目文件、文件夹及完整修订历史的单元，分为公共（Public）和私有（Private）     两种，可理解为"代码项目文件夹"。

2. 分支（Branch）：从主代码线分离出的独立开发线路，允许在不影响主分支（通常为 main）的前提下开发功能或修复漏洞，是协作的核心基础。
3. 提交（Commit）：对暂存区更改的快照保存，包含修改内容、作者、时间等信息，提交信息需清晰描述更改（避免"fix bug"等模糊表述）。
4. 拉取请求（Pull Request, PR）：向仓库维护者提议合并分支更改的功能，可在合并前进行代码讨论与审查，推荐使用"Squash and merge"模式合并。
5. .gitignore 文件：指定 Git 无需跟踪的文件/文件夹（如依赖包、敏感配置），建议设置全局 .gitignore 统一管理通用规则。
